# agent-setup

My personal [pi](https://www.npmjs.com/package/@earendil-works/pi-coding-agent)
agent configuration — the parts I actually authored — kept under version control
so I can reproduce my setup on any machine (Linux, macOS, WSL, or Windows).

## What's in here

This repo lives at `~/.pi` (the pi agent config directory). It **only** tracks
self-authored config; everything installed or generated by pi is git-ignored.

| Path | What it is |
|------|------------|
| `install-pi-extensions.mjs` | Cross-platform bootstrap installer (pure Node, no deps) |
| `agent/settings.json` | pi settings (provider, model, packages, themes, skills) |
| `agent/prompts/` | Custom prompt templates |
| `agent/extensions/aws-sso-auto-relogin.ts` | Self-authored extension: auto AWS SSO re-login |
| `agent/extensions/slim-ui.ts` | Self-authored extension: slimmer TUI |

Everything else under `~/.pi` (installed extensions, packages, skills, sessions,
`auth.json`, backups, etc.) is intentionally **not** tracked — it gets restored
by the installer below.

## Install on a fresh machine

```bash
git clone git@github.com:wapacz/agent-setup.git .pi && node .pi/install-pi-extensions.mjs
```

> Run it from your home directory so the repo lands at `~/.pi`.
> On Windows that resolves to `%USERPROFILE%\.pi`.

`install-pi-extensions.mjs` then:

1. **Downloads extensions** into `agent/extensions/` — directories via the GitHub
   API (runs `bun install` if they ship a `package.json`), single `.ts` files
   via `raw.githubusercontent.com`.
2. **Installs pi packages** via `pi install` (providers, themes, subagents, …).
3. **Installs skills globally** via `npx skills add … --global`, skipping any
   already installed, then runs `npx skills update --global`.

### Requirements

- **Node.js ≥ 18** (built-in `fetch`) — required
- **pi** on `PATH` — for the `pi install` step (skipped with a warning if missing)
- **bun** — only for extensions that ship a `package.json` (skipped with a warning if missing)
- Optional: `GITHUB_TOKEN` env var to raise the GitHub API rate limit (60/h anonymous)

## Usage

```bash
# Full built-in run: extensions + pi packages + skills
node install-pi-extensions.mjs

# Only install specific extensions (skips pi packages & skills)
node install-pi-extensions.mjs notify fetch
node install-pi-extensions.mjs disler/pi-vs-claude-code:extensions/minimal.ts
```

### Extension entry format

```
[<owner>/<repo>:]<path>[=<name>]
```

- `<path>` — location inside the repo; a directory (`notify`, `extensions/notify`)
  or a single `.ts` file (`extensions/minimal.ts`)
- omit `<owner>/<repo>:` to use the default repo (`rytswd/pi-agent-extensions`)
- append `=<name>` to override the install name (e.g. to avoid collisions)

Edit the `EXTENSIONS`, `PI_PACKAGES` and `SKILLS` arrays at the top of
`install-pi-extensions.mjs` to change what the built-in run installs.

## Notes

- No secrets are committed. `agent/auth.json` and everything sensitive is
  git-ignored. Git history was intentionally started fresh to keep it clean.
